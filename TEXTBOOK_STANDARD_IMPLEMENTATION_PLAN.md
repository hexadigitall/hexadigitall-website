# Hexadigitall Advanced Textbook Standard

This document defines the canonical standard for advanced Hexadigitall textbooks.

It is no longer just a gap-analysis note. It is the working product standard to be used when creating, upgrading, or reviewing any advanced textbook in the line.

The standard combines the strongest product and mentorship patterns from the Architecting Landing Zones book with the strongest delivery, assessment, edition, and print patterns from the DevOps books.

Operational publishing handoff:

Once a textbook has been created or upgraded to meet this standard, move to [TEXTBOOK_PRINT_AND_SALES_PLAYBOOK.md](/mnt/d/projects/mine/hexadigitall/hexadigitall/TEXTBOOK_PRINT_AND_SALES_PLAYBOOK.md) for the end-to-end publishing pipeline covering derived manuscripts, covers, full-wrap designs, PDF generation, and marketplace release.

## Standard Intent

Every advanced Hexadigitall textbook must work as all of the following at once:

1. a premium textbook product
2. a mentorship delivery system
3. a cohort/course delivery manual
4. a portfolio-building workbook
5. a print-ready publication
6. a digital-first learning asset

The standard assumes that a title may be sold standalone, used in mentorship, used in a live course, or distributed as part of private enterprise delivery.

This document defines what the textbook product must become. The print-and-sales playbook defines how that finished textbook product becomes a commercial asset family.

## Core Design Principle

The canonical advanced textbook is the fusion of two proven strengths:

1. Landing Zones contributed premium product framing, mentorship pathways, executive-delivery positioning, strong front matter, resources/errata paths, and premium commercial structure.
2. DevOps contributed repeatable instructional mechanics, teacher/student edition alignment, weekly assessment architecture, troubleshooting flow, rubric discipline, back-cover polish, and interactive textbook assessment support.

All future advanced textbooks should inherit both strengths together.

## Canonical Product Model

Each advanced textbook must be intentionally positioned as:

1. a serious technical manual
2. a premium Hexadigitall product
3. a guided professional transformation path
4. a portfolio evidence system
5. a job-readiness or client-readiness asset

This means the book must not feel like lecture notes only, and it must not feel like a marketing brochure only. It must be commercially credible and operationally useful at the same time.

## Canonical Textbook Stack

Every advanced textbook should be built in the following layers.

### 1. Packaging And Brand Layer

Required:

1. premium front cover
2. strong subtitle communicating track, level, and scope
3. front-cover metadata block with duration, level, format, prerequisites, and track
4. QR code linking to course or textbook support path
5. Hexadigitall brand URL
6. polished back cover with summary, audience, outcomes, publisher details, and barcode/ISBN placeholder
7. consistent premium typography and color discipline

Proven patterns:

1. Landing Zones proves the premium flagship presentation style.
2. DevOps proves the front-cover metadata block and robust back-cover structure.

### 2. Commercial And Access Layer

Required:

1. license and access statement
2. purchaser usage terms
3. resources URL
4. errata and updates URL
5. mentorship URL
6. course/cohort URL
7. support window or support expectations
8. corporate/private delivery contact path where relevant

This is mandatory for sale-readiness.

### 3. Mentorship And Enrollment Layer

Required:

1. explicit dual-pathway framing
2. Pathway A: modular mentorship or monthly support structure
3. Pathway B: full course or cohort experience
4. statement of when mentorship is strongly recommended
5. enrollment CTA
6. cohort CTA
7. enterprise/private cohort CTA where relevant

Proven patterns:

1. Landing Zones established the mentorship-first product framing.
2. DevOps confirmed the same pattern can be carried into teacher and student books consistently.

### 4. Front Matter Layer

Required core pages:

1. copyright and access page
2. license and updates page
3. mentorship pathways and enrollment page
4. table of contents
5. how to use this textbook
6. methodology or learning framework page
7. prerequisites and preparation page
8. success strategies page
9. student guidance page
10. instructor guidance page in teacher editions

Recommended front matter additions where relevant:

1. author or institution note
2. positioning statement or preface
3. course preparation and mentorship workflow guide
4. portfolio repository rules page
5. career outcome framing
6. infrastructure and cloud-cost disclaimer page

### 5. Weekly Or Unit Architecture Layer

3. learning objectives
4. pre-class or pre-reading tasks
5. detailed teaching notes
6. labs or implementation tasks
7. solutions and troubleshooting guidance
8. practice questions
9. weekly quiz
10. weekly test or applied scenario assessment
11. weekly assignment
12. rubric or evaluation guidance
13. portfolio evidence expectations
14. mentorship callout
15. reflection or review prompt

Student edition canonical order:

1. week opener or chapter opener
2. strategic context and why the week matters
3. learning objectives
4. pre-class or pre-reading tasks
5. main learning content

7. student-safe troubleshooting guidance
8. practice questions
9. weekly quiz
10. weekly test or applied scenario assessment
11. weekly assignment
12. portfolio evidence expectations
13. mentorship callout
14. reflection or review prompt

Student editions must not expose instructor-only answer keys, grading internals, or facilitation-only notes.

### 6. Portfolio Evidence Layer

### 7. Assessment Layer

Required:

1. practice questions
2. weekly quiz
3. weekly test or scenario assessment
4. weekly assignment

1. quizzes are formative

Required where the book includes hands-on implementation:

Recommended components:

2. iterative mastery
3. problem-based learning
4. experiential practice
5. scaffolded difficulty
6. continuous assessment

### 10. Capstone And Career Layer

Required:

2. capstone outputs defined clearly
3. defense, walkthrough, or handover expectation
4. interview or executive presentation framing where relevant

Landing Zones proves the strongest capstone framing with executive handover and architecture defense.

DevOps proves the strongest portfolio and applied-assessment continuity into weekly work.

## Teacher And Student Edition Rules

Every advanced title with dual editions must obey the following boundaries.

### Shared Between Teacher And Student Editions

Required shared layers:

1. front cover identity
2. commercial and access layer
3. mentorship pathways
4. front matter sequence
5. portfolio rules
6. capstone narrative
7. weekly learning sequence
8. print system and brand system

### Teacher-Only Material

Teacher editions may include:

1. answer keys
2. grading rubrics
3. facilitation notes
4. teaching notes
5. instructor workflow guidance
6. private evaluation criteria
7. answer visibility inside embedded assessments

### Student-Safe Material

Student editions must:

1. preserve the same premium structure and learning flow
2. preserve assessments in a usable learning format
3. hide instructor-only answers when assessments are embedded
4. remove private grading internals
5. remove instructor-only facilitation logic

## Interactive Assessment Standard

This standard is now part of the advanced textbook system.

When a textbook includes embedded assessments:

1. assessment definitions must be keyed by the correct course slug
2. the website must support timed exam mode separately from textbook study mode
3. textbooks may embed study-mode assessments directly in the textbook experience
4. teacher editions may show answers where pedagogically appropriate
5. student editions must be able to hide answers while preserving the full assessment content
6. links-only is not sufficient when the product requirement is an embedded textbook assessment

Implementation pattern proven in DevOps:

1. shared assessment registry keyed by canonical slug
2. per-course assessment definition file
3. public endpoint for textbook assessment payloads
4. shared textbook assessment renderer
5. data-attribute control for showing or hiding answers per textbook variant

## Print And Layout Standard

This is now a canonical requirement, not an optional polish item.

### Required Print Baseline

1. advanced books should default to 6in x 9in trim unless there is an explicit publishing exception
2. cover and back cover must render as single-page layouts without spill
3. front matter and main matter must use a unified print system
4. page-break behavior must be deliberate and stable
5. callouts, code blocks, diagrams, and figures should avoid ugly splits where possible

### Current Working 6x9 Interior Guidance

Based on the DevOps fixes, the current standard direction is:

1. true 6x9 page box
2. mirrored gutter margins for left and right pages
3. tighter outside margins than previous A4-derived layouts
4. content padding tuned specifically for 6x9, not desktop-screen spacing
5. list indentation reduced for narrow-page efficiency
6. cover layout using constrained flex structure rather than oversized absolute positioning

### Print Anti-Patterns To Avoid

1. leaving `@page` at A4 while claiming 6x9 output
2. reusing desktop paddings in print chapters
3. oversized cover spacing designed for taller pages
4. teacher and student books drifting into different print systems without reason
5. hardcoding manual page number footers or spans instead of using CSS-based pagination with `@page` rules and `counter(page)`
6. mixing embedded numeric footer elements with print-engine pagination (causes duplicate or incorrect page numbering)
7. assuming visible page numbers in browser view will match final PDF output without CSS-driven pagination
8. using `0.375in` for hardcover full-wrap hinges — the correct KDP case laminate hinge is `0.0625in`; oversized hinges compress the front panel and make the spine appear narrower than it is
9. generating JPG or TIFF from full-wrap cover files — full-wrap formats are print-press PDFs only
10. generating Kindle cover images via headless browser screenshot — render to PDF first, then convert with ImageMagick to get correct inch-based dimensions

**Pagination guidance:**

All advanced textbooks must delegate page numbering entirely to the CSS print engine via `@page` rules and `counter(page)` mechanisms. Do not hardcode numeric page footers as HTML span elements. This ensures that the final PDF page count and numbering are generated by the print engine, not by conflicting manual elements embedded in the source content.

## Implementation Standards For Future Books

When building a new advanced book or upgrading an existing one, follow this order.

### Phase 1. Product Skeleton

1. define title, track, level, duration, and product positioning
2. define course, mentorship, resources, and errata URLs
3. define teacher and student edition strategy
4. define repository naming and artifact rules

### Phase 2. Brand And Print System

1. build front cover
2. build back cover
3. apply 6x9 print system
4. validate page breaks and cover fit

### Phase 3. Front Matter System

1. license and access page
2. mentorship and enrollment page
3. how-to-use page
4. methodology page
5. prerequisites and risk disclaimers
6. portfolio guidance

### Phase 4. Weekly Or Unit Learning System

1. standardize week opener structure
2. standardize labs and troubleshooting
3. standardize assessment sequence
4. standardize portfolio output expectations
5. add mentorship callouts where relevant

### Phase 5. Dual Edition Controls

1. preserve shared product structure
2. isolate teacher-only content
3. ensure student-safe answer visibility
4. verify parity of print quality and front matter

### Phase 6. Interactive Assessment Integration

1. confirm canonical course slug
2. register assessments consistently
3. expose study-mode payloads for textbook embedding
4. configure student and teacher answer visibility correctly
5. verify site exam mode remains separate from textbook embedding

## Acceptance Checklist

Use this before calling any advanced textbook ready.

### Brand And Packaging

1. front cover is premium and on-brand
2. back cover is complete and polished
3. metadata block is present
4. QR and URL paths are valid

### Commercial Readiness

1. mentorship pathways are explicit
2. enrollment CTA is present
3. resources path is present
4. errata path is present
5. support expectations are present

### Front Matter Quality

1. how-to-use guidance is present
2. methodology framing is present
3. prerequisites are clear
4. cost and access warnings are present where needed
5. portfolio rules are explicit

### Delivery Quality

1. week structure is consistent
2. labs are operationally usable
3. troubleshooting exists where relevant
4. assessments are structured clearly
5. capstone is framed professionally

### Edition Control

1. teacher and student editions share the same product standard
2. teacher-only material is not leaking into student editions
3. embedded assessments respect answer visibility rules

### Publishing Quality

1. page size is truly 6x9 if that trim is claimed
2. text block is appropriate for 6x9, not inherited from A4
3. cover and back cover do not spill
4. page-break behavior is stable
5. no hardcoded page number footers or numeric spans are embedded in the source (all pagination is CSS-generated)
